# PrintAutoTestcases 开发规范文档

本目录包含项目的所有开发标准流程规范文档，确保团队开发的一致性和代码质量。

## 📋 规范文件列表

### 1. 前端开发规范
**文件**: `前端开发rules.mdc`
- React 18 + TypeScript + Ant Design 开发标准
- 组件开发规范和最佳实践
- 状态管理、API调用、性能优化
- 测试覆盖率要求和开发流程

### 2. 后端开发规范  
**文件**: `backend_dev_rules.mdc`
- FastAPI + SQLAlchemy + Pydantic 开发标准
- API设计规范和错误处理
- 数据库操作和服务层架构
- 安全性规范和异步任务处理

### 3. UI开发规范
**文件**: `ui_dev_rules.mdc`
- 现代化企业级设计系统标准
- 设计Token、颜色、字体、间距规范
- 组件库使用和交互设计标准
- 响应式设计和无障碍设计要求

### 4. BUG修复规范
**文件**: `bug_fix_rules.mdc`
- 六阶段BUG修复流程（确认→分析→设计→修复→测试→监控）
- 问题定位方法和根因分析
- 修复方案设计和代码审查要点
- 部署监控和预防措施

## 🚀 如何使用

### Cursor AI 自动应用
这些规范文件已配置为 Cursor AI 的开发规范，在生成代码时会自动遵循：

1. **前端代码生成** - 自动应用React + TypeScript + Ant Design规范
2. **后端代码生成** - 自动应用FastAPI + SQLAlchemy规范  
3. **UI组件开发** - 自动应用设计系统和交互规范
4. **BUG修复** - 自动应用标准修复流程

### 团队开发使用
1. **新人入职**: 必须阅读所有规范文档
2. **代码开发**: 严格按照对应模块的规范执行
3. **代码审查**: 使用规范文档作为审查标准
4. **BUG修复**: 必须遵循六阶段修复流程

## ✅ 规范要点速查

### 命名规范
- **前端**: PascalCase文件名、camelCase变量、UPPER_SNAKE_CASE常量
- **后端**: snake_case文件名和函数、PascalCase类名

### 质量要求
- **测试覆盖率**: ≥ 80%
- **代码审查**: 必须2人以上审查
- **错误处理**: 前端try-catch + 后端HTTPException

### 开发流程
1. 创建功能分支
2. 遵循编码规范
3. 编写测试用例
4. 代码审查
5. 合并部署

2. 团队评审和决策
