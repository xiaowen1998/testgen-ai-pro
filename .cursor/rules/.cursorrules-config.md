# Cursor Rules 配置说明

## 文件结构说明

```
.cursor/
├── rules/
│   ├── README.md                    # 规范文档总览
│   ├── .cursorrules-config.md      # 本配置说明文件
│   ├── 前端开发rules.mdc           # 前端开发规范（自动应用）
│   ├── backend_dev_rules.mdc       # 后端开发规范（自动应用）
│   ├── ui_dev_rules.mdc           # UI开发规范（自动应用）
│   └── bug_fix_rules.mdc          # BUG修复规范（自动应用）
└── .cursorrules                    # 主配置文件
```

## 自动应用机制

### 1. 文件类型检测
Cursor会根据当前编辑的文件类型自动选择对应的规范：

- **`.tsx`, `.ts`, `src/components/`, `src/pages/`** → 应用前端开发规范
- **`.py`, `backend/app/`, `*/api/`** → 应用后端开发规范  
- **UI组件、设计相关** → 应用UI开发规范
- **BUG修复、问题排查** → 应用BUG修复规范

### 2. 上下文识别
Cursor会分析代码上下文和用户意图：

- 创建新组件 → 使用组件模板和前端规范
- 编写API接口 → 使用API模板和后端规范
- 修复错误 → 应用BUG修复流程
- 设计界面 → 应用UI设计规范

### 3. 质量检查
生成代码时自动进行质量检查：

- ✅ 命名规范检查
- ✅ 类型注解检查  
- ✅ 错误处理检查
- ✅ 文档注释检查
- ✅ 代码结构检查

## 使用最佳实践

### 📝 代码生成
在Cursor中直接描述需求，AI会自动应用对应规范：

```
// 用户请求示例
"创建一个用户登录组件，包含邮箱和密码输入框"
→ 自动应用前端开发规范，生成符合Ant Design和TypeScript规范的组件

"创建用户登录API接口"  
→ 自动应用后端开发规范，生成符合FastAPI和Pydantic规范的接口

"修复登录功能的bug"
→ 自动应用BUG修复规范，按六阶段流程进行修复
```

### 🔧 代码重构
选中代码片段，描述重构需求：

```
"将这个组件改为使用Ant Design的Form组件"
→ 自动应用前端规范，确保符合Ant Design设计标准

"优化这个API的错误处理"
→ 自动应用后端规范，统一错误响应格式
```

### 🐛 问题修复
描述遇到的问题：

```
"用户无法登录，显示404错误"
→ 自动应用BUG修复规范：
  1. 确认问题（收集错误信息）
  2. 分析根因（检查API和数据库）
  3. 设计方案（选择最佳修复方案）
  4. 实施修复（遵循代码规范）
  5. 测试验证（确保修复有效）
  6. 监控部署（防止回归问题）
```

## 配置优化建议

### 1. 团队协作
- 所有团队成员使用相同的规范配置
- 定期review和更新规范内容
- 新人入职时必须熟悉所有规范

### 2. 持续改进
- 收集使用反馈，优化规范内容
- 根据项目发展调整规范标准
- 保持规范的时效性和实用性

### 3. 质量监控
- 代码review时检查规范遵守情况
- 定期进行代码质量审计
- 建立规范违反的纠正机制
